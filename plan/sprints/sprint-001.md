# Sprint 001: 렌더링 엔진 & 스프라이트 생성

## 목표
게임 캔버스가 화면에 표시되고 프레임 루프가 동작하며, 프로시저럴 몬스터 스프라이트가 생성되어 렌더링된다.

## 대상 기능
- F-001: 게임을 시작하면 화면에 캔버스가 표시되고 게임 루프가 동작한다
- F-020: 프로시저럴 픽셀아트로 102종 몬스터 스프라이트가 생성된다

## 선행 조건
없음 (첫 스프린트)

## 작업 항목
- [ ] HTML 페이지에 800x600 캔버스 요소를 배치한다
- [ ] requestAnimationFrame 기반 게임 루프를 구현한다 (update + render 분리)
- [ ] 상태 머신 기본 골격을 구현한다 (LOADING → TITLE)
- [ ] 커스텀 5x7 비트맵 폰트 렌더링 함수를 구현한다
- [ ] 기본 렌더링 유틸리티를 구현한다 (사각형, 선, 텍스트, 색상)
- [ ] 프로시저럴 스프라이트 생성기를 구현한다 (색상 팔레트, 형태 템플릿, 특징 조합)
- [ ] monsters.json 데이터에서 spriteConfig를 읽어 고유 스프라이트를 생성한다
- [ ] 스프라이트 캐시 시스템을 구현한다
- [ ] 타이틀 화면을 표시한다 (게임 제목, 별 애니메이션 배경)

## 완료 조건 (Done Criteria)
- [ ] 브라우저에서 800x600 캔버스가 표시된다 (검증 방법: Playwright로 canvas 요소 존재 확인)
- [ ] 게임 루프가 60fps로 동작한다 (검증 방법: requestAnimationFrame 카운트 측정, 50fps 이상)
- [ ] 타이틀 화면에 게임 제목 텍스트가 렌더링된다 (검증 방법: 스크린샷에서 단색이 아닌지 확인)
- [ ] 최소 3종 이상 몬스터 스프라이트가 서로 다른 색상/형태로 생성된다 (검증 방법: 픽셀 비교)
- [ ] 스프라이트 캐시가 동작하여 동일 몬스터 재생성이 발생하지 않는다 (검증 방법: 콘솔 로그)

## 예상 파일 구조
```
index.html
src/main.js              # 게임 클래스, 메인 루프, 상태 머신
src/ui/renderer.js       # 캔버스 렌더링 엔진, 폰트, 유틸리티
src/ui/sprite-generator.js  # 프로시저럴 스프라이트 생성
data/monsters.json       # (기존) 몬스터 데이터 참조
vite.config.js           # Vite 설정
```
