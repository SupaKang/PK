# Sprint 009: 아이템 / 상점 / 세이브

## 목표
42종 아이템을 상점에서 구매/판매하고, 3슬롯 세이브/로드가 동작한다.

## 대상 기능
- F-010 일부: 아이템/상점, 세이브/로드

## 에셋 요구사항
- 상인 NPC (4방향, idle)
- 상점 UI 프레임 (256x192)
- 인벤토리 그리드 셀 (32x32)
- 상점 건물 맵 오브젝트

## 선행 조건
Sprint 008 산출물 (스토리 진행, 전체 게임 루프)

## 작업 항목
- [ ] PixelLab으로 상인 NPC + 상점 UI + 상점 건물 생성
- [ ] 인벤토리 시스템 (아이템 목록, 수량, 카테고리)
- [ ] items.json 데이터 로드 및 효과
- [ ] 상점 UI (구매/판매, 가격, 잔고)
- [ ] 필드 아이템 사용 (메뉴에서 회복)
- [ ] 세이브 시스템 (파티/위치/인장/카르마/인벤토리/스토리 직렬화)
- [ ] LocalStorage 3슬롯 저장/불러오기
- [ ] 마을 진입 시 자동 저장
- [ ] FSM에 SHOP 상태 추가

## 완료 조건 (Done Criteria)
- [ ] 상점 구매 → 인벤토리 추가 + 골드 차감 (검증: 전후 비교)
- [ ] 전투 중 물약 → HP 회복 (검증: HP 비교)
- [ ] 저장 → 새로고침 → 로드 → 상태 복원 (검증: 저장-로드 사이클)
- [ ] 3슬롯 독립 동작 (검증: 각 슬롯 별도 데이터)

## 예상 파일 구조
```
src/core/item.js            # 인벤토리, 아이템 효과
src/save/save.js            # 세이브/로드
src/ui/menu-ui.js           # (수정) 상점 UI, 세이브 UI
src/assets/sprites/npcs/    # 상인 에셋
src/assets/ui/              # 상점 UI 에셋
src/assets/objects/         # 상점 건물 에셋
```
