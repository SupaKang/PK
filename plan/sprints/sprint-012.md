# Sprint 012: 모바일 터치 / 통합 QA

## 목표
모바일 터치로 모든 기능을 조작할 수 있고, 전체 게임 플로우가 시작부터 엔딩까지 크래시 없이 동작한다.

## 대상 기능
- F-012 일부: 모바일 터치 대응

## 에셋 요구사항
- 모바일 D-패드 (96x96, 반투명)
- A/B 버튼 (48x48, 반투명)

## 선행 조건
Sprint 011 산출물 (모든 기능 완성)

## 작업 항목
- [ ] PixelLab으로 D-패드/A/B 버튼 에셋 생성
- [ ] 터치 입력 시스템 (가상 D-패드, 액션 버튼)
- [ ] 반응형 캔버스 스케일링
- [ ] 터치 기반 메뉴/대화/상점 조작
- [ ] PWA manifest
- [ ] 전체 E2E 통합 테스트:
  - 타이틀 → 프롤로그 → 맵 → 전투 → 포획
  - 체육관 → 인장 → 지역 해금
  - 스토리 → 라이벌 → 보스
  - 사천왕 → 마왕 → 엔딩
  - 세이브/로드, 도감/업적
- [ ] 성능 최적화 (60fps, 번들 250KB 이하)
- [ ] 크로스 브라우저 테스트

## 완료 조건 (Done Criteria)
- [ ] 모바일 D-패드 터치로 이동 (검증: 터치 이벤트 테스트)
- [ ] 전투 스킬 선택 터치 동작 (검증: 터치로 전투 진행)
- [ ] 전체 플로우 크래시 없음 (검증: E2E 시나리오)
- [ ] 60fps 유지 (검증: FPS 카운터 50fps 이상)
- [ ] 번들 250KB 이하 (검증: 빌드 크기)
- [ ] 터치 UI가 PixelLab 에셋 (검증: src/assets/ui/ 파일 존재)

## 예상 파일 구조
```
src/ui/touch-controls.js    # 터치 입력, D-패드
src/main.js                 # (수정) 반응형 스케일링
public/manifest.json        # PWA
src/assets/ui/              # 터치 컨트롤 에셋
```
