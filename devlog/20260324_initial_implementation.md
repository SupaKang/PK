# 개발 로그 — Pocket Kingdom 초기 구현
- 날짜: 2026-03-24
- 프로젝트: PK (턴제 몬스터배틀 RPG)

## 개요
포켓몬스터를 오마주한 웹 기반 턴제 몬스터배틀 RPG "Pocket Kingdom" 전체 초기 구현 완료.

## 기술 스택
- **프레임워크**: Vanilla JS + HTML5 Canvas + Vite
- **렌더링**: Canvas 2D API (800x600, pixelated)
- **데이터**: JSON 정적 파일 (public/data/)
- **세이브**: LocalStorage 3슬롯
- **오디오**: Web Audio API 프로시저럴 칩튠
- **스프라이트**: 프로시저럴 픽셀아트 생성 (64x64)

## 구현된 시스템

### 코어 엔진 (src/core/)
| 파일 | 기능 |
|---|---|
| `type.js` | 18타입 상성 시스템 (화염/해류/자연/전격/대지/빙결/질풍/맹독/강철/광명/암흑/영혼/용린/투지/암석/음파/우주/무속성) |
| `monster.js` | 몬스터 생성, 스탯 계산 (IV/EV), 레벨업, 진화, 스킬 학습 |
| `skill.js` | 데미지 공식 (포켓몬 기반), STAB, 크리티컬, 상태이상 (화상/독/마비/수면/혼란/빙결) |
| `battle.js` | 턴제 배틀 엔진 — AI 행동결정, 선공 판정, 포획 시스템 |
| `party.js` | 6마리 파티 + 보관함 관리 |
| `capture.js` | 포획 확률 계산 (HP 비례, 볼 배율) |
| `item.js` | 아이템/인벤토리/상점 시스템 |

### 월드 시스템 (src/world/)
| 파일 | 기능 |
|---|---|
| `map.js` | 노드 기반 월드맵 (8마을, 7루트, 8체육관, 2동굴, 리그) |
| `encounter.js` | 야생 인카운터 (가중 랜덤, 레벨 범위) |
| `npc.js` | 트레이너 관리 (패배 기록, 팀 생성) |
| `story.js` | 9챕터 스토리 진행 (이벤트, 배지, 챕터 전환) |

### UI 시스템 (src/ui/)
| 파일 | 기능 |
|---|---|
| `sprite-generator.js` | 10종 형태(bipedal/quadruped/serpent/avian/amorphous/insectoid/aquatic/humanoid/mythical/elemental) + 10종 특징 조합으로 프로시저럴 픽셀 스프라이트 생성 |
| `renderer.js` | 내장 5x7 비트맵 폰트, RPG 스타일 패널, HP/EXP 바, 화면 흔들림 |
| `battle-ui.js` | 배틀 화면 — 타이프라이터 메시지, HP 애니메이션, 스프라이트 슬라이드, 데미지 플래시 |
| `menu-ui.js` | 타이틀(별 애니메이션), 게임메뉴(파티/가방/도감/트레이너카드/저장/설정), 상점, 파티뷰 |
| `dialog-ui.js` | 대화창 — 타이프라이터, 선택지, 초상화 |
| `map-ui.js` | 월드맵 — 미니맵 노드 레이아웃, 위치정보 패널, 이동 트랜지션 |
| `dex-ui.js` | 도감 — 10x10 그리드, 상세뷰(스탯 바 차트), 발견/포획 추적 |

### 인프라
| 파일 | 기능 |
|---|---|
| `save/save.js` | LocalStorage 3슬롯 세이브/로드, 슬롯 미리보기 |
| `audio/audio.js` | 5 BGM(배틀/마을/루트/보스/승리) + 9 SFX, 프로시저럴 칩튠 |
| `main.js` | 게임 루프, 상태 머신, 시스템 통합 |

## 게임 데이터

### 몬스터 (100종)
- **3단 진화 15체인** (45종): 스타터 3종(화염/해류/자연) + 12종(전격/빙결/질풍/맹독/강철/광명/암흑/영혼/용린/투지/암석/음파)
- **2단 진화 20체인** (40종): 혼합 타입
- **단독 5종**: 진화 없는 일반 몬스터
- **전설 10종** (ID 91-100): 태양신수, 심연해왕, 세계수, 뇌제, 시간지배자, 공간창조자 등

### 스토리 (9챕터)
1. 여행의 시작 — 박사, 스타터 선택, 라이벌 첫 배틀
2. 첫 번째 배지 — 풀빛 체육관
3. 그림자단의 그림자 — 동굴 이벤트
4. 물의 도시 — 라이벌 재대결
5. 폭풍의 전조 — 그림자단 간부
6. 사막의 비밀 — 전설 몬스터 힌트
7. 얼어붙은 진실 — 라이벌 3차전
8. 최후의 결전 — 그림자단 보스전 + 리그
9. 새로운 챔피언 — 엔딩

### 체육관 (8개)
| 순서 | 이름 | 타입 | 관장 | 레벨대 |
|---|---|---|---|---|
| 1 | 풀빛 체육관 | 자연 | 초록 | 12-14 |
| 2 | 암석 체육관 | 암석 | 석진 | 17-19 |
| 3 | 파도 체육관 | 해류 | 미나 | 23-25 |
| 4 | 번개 체육관 | 전격 | 라이 | 28-30 |
| 5 | 투지 체육관 | 투지 | 무현 | 33-35 |
| 6 | 빙하 체육관 | 빙결 | 설화 | 38-41 |
| 7 | 용린 체육관 | 용린 | 용왕 | 44-48 |
| 8 | 우주 체육관 | 우주 | 성운 | 49-53 |

## 데이터 생성 방식
- `scripts/generate-data.mjs` — Node.js 스크립트로 monsters.json(100종), maps.json, story.json 프로시저럴 생성
- `types.json`, `skills.json`, `items.json` — 에이전트가 직접 작성

## 빌드 & 실행
```bash
npm install
npm run dev          # 개발 서버 (http://localhost:8080)
npm run build        # 프로덕션 빌드 → dist/
npm run generate-data  # 몬스터/맵/스토리 데이터 재생성
```
